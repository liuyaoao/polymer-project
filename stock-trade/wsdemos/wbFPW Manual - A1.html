Appendix I: What Is MATLAB?

 

Financial PatternWorkshop is built on the platform of MATLAB.  The name MATLAB stands for matrix laboratory. In university environments, it is the standard instructional tool for introductory and advanced courses in mathematics, engineering, and science. In industry, MATLAB is extremely popular. Over half a million users have found that the combination of the intuitive MATLAB interface, language, and the built-in math and graphics functions make MATLAB the preferred platform for technical computing over C, Fortran, and other languages and applications.

 

MATLAB integrates computation, visualization, and programming in an easy-to-use environment where problems and solutions are expressed in familiar mathematical notation.  With more than 600 mathematical, statistical, and engineering functions, MATLAB gives one immediate access to high-performance numeric computing. The numerical routines are fast, accurate, and reliable. In addition, MATLAB lets one integrate external routines written in C, C++, Fortran, and Java with MATLAB applications.  Combining all of the above benefits makes it clear why we choose MATLAB to develop our Financial PatternWorkshop.  Financial data are just matrix composed of open, high, low, close and volume no matter what your time scale is.  One does not need to learn MATLAB to use FPW as long as one knows some most basic mathematical operators like +, -, *, / and some program flow control commands, like if, else, elseif, while and for.   Following our examples in the manual, one will gradually learn how to put MATLAB code into FPW.  But understanding the following Matlab basics will be very helpful for using FPW.

 

Matrix:  Informally, the terms matrix and array are often used interchangeably. More precisely, a matrix is a two-dimensional rectangular array of real or complex numbers that represents a linear transformation. The element in row i and column j of matrix A is denoted by A(i,j).  Subscript expressions involving colons refer to portions of a matrix.  For example, A(1:4,j) means the first 4 elements of the j-th column of A, and A(i,:) means the all the elements in the i-th row of A. For FPW users, the data we actually deal with are two-dimensional arrays. Each row contains what Wall Street calls bar information, like open, high, low, close, volume, net, etc. And each column contains the history of its each row member.  FPW data format is that the last row is the most recent bar data.

 

M-file: Files that contain Matlab language code are called m-files. Its file name extension is ".m".  M-files can be functions that accept arguments and produce output, or they can be scripts that execute a series of Matlab statements.  One can create m-files by using any kind of text editor, then use them as one would any other Matlab function or command.  For FPW users, the Matlab language code you enter will be automatically saved as an M-file by FPW.

 

Local and Global Variables: Ordinarily, each Matlab function, defined by an M-file, has its own local variables, which are separate from those of other functions, and from those of the base workspace.  However, if several functions, and possibly the base workspace, all declare a particular name as global, then they all share a single copy of that variable.  Any assignment to that variable, in any function, is available to all the other functions declaring it global.  For FPW users, one will use dozens of global variables, which are declared by FPW and have set specific meanings.  For examples: wstime - signal triggering time, lotshis - position share or lots history.

 

Operators: Matlab’s operators fall into three categories: Arithmetic operators, relational operators and logical operators. Arithmetic operators have the highest precedence, then relational operators, and then logical operators. One can use pair bracket of ‘(’ and ‘)’ to break above precedence orders.  Arithmetic operators perform numeric computations.  They are (in the order of their precedence): transpose (.’), power (.^), plus (+), minus (-), multiplication (.*), division (./), multiplication (*), division (/) and colon operator (:).  The notation (.) in the operators means to do the operation to the each element of the operand matrix or array.  Relational operators compare operands quantitatively. They are less than (<), less than or equal to (<=), greater than (>), greater than or equal to (>=), equal to (==) and not equal to (~=).  Logical operators implement the operations AND (&), OR ( | ) and NOT (~), with the results corresponding to TRUE (1) or FALSE (0).

 

Flow control: There are four flow control statements in Matlab.

1, if, together with else and elseif, executes a group of statements based on some logic conditions.  Its basic

    form is:

                if expression A

                   statements block A

                elseif expression B

                   statements block B

                                .

                                . (can have more than one elseif block)

                else

                   statements block  C

                end

    Where the 3rd to the 8th lines can be partially or all omitted.

 

2, switch, together with case and otherwise, executes a group of statements based on some logic conditions.

    Its syntax is

                switch expression (scalar or string)

                   case value 1

                      statements 1

                   case value 2

                      statements 2

                                .

                                . (can assigned as many cases as one likes)

                   otherwise

                       statements N

                    end

 

3, while executes a group of statements an indefinite number of times, based on some logic conditions.

    Usually uses break to break the indefinite loop.  Its syntax is

                while expression

                    statements

                end

   If a while condition reduces to an empty array, statements will be never executed.

 

4, for executes a group of statements a fixed number of times.  Its syntax is

                for index = beging_number:increment:end_number

                     statements

                end

For FPW users, the first and the last ones are the most useful.

 

Long Command lines:  If a statement does not fit on one line, use three periods, (...), followed by return or enter to indicate that the statement continues on the next line.  Blank spaces around the =, +, - and other operators are optional, but they improve readability.  The maximum number of characters allowed on a single line is 4096.

 

Some names you should not use when you name your variables.  They are help, who, whos, what, type, dir, cd, which, edit, end, for, if, case, switch, else, elseif, otherwise, size, all, sort, rand, save, load, exit, break, stop, short, long, pack, min, max, mean, std, input, disp, abs, any, ans, cos, sin, inf, norm, pi and others with obvious meaning because all these are MATLAB functions or part of them. Matlab does not prohibit you to use them, but you’d better use them as what they are.  In addition, FPW users will also learn that dozens of global variables and function names of FPW are pre-decided by the program and you must use them accordingly, please see the sections of Variables and Functions.

 

Finally, in FPW, commands and variables are case sensitive. One should use all Matlab and FPW command in lower case unless specifically required.
