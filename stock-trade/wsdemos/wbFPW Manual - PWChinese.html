什么是模式站？ 

模式站是一个综合软件包，操作人员通过输入几行简单的程序及或点击几下鼠标则可对股票及期货等金融市场的各种复杂交易系统进行模拟。操作人员针对股票及期货市场几乎可以模拟想象中的任何由各种规则组成的模式和由各种入场、离场方式(如逐步渐入渐出)、各种止损及各种交易目标组成的管理方式的交易系统，操作人员可以一次针对一个或一组市场用相同或不同的资金管理方式进行模拟。

 

进入帐号后，点击PatternWorkshop即可进入模式站，其操作界面如图1.1所示。它由四大部分组成。它们分别是：在右下角的市场输入界面(图1.2)，在左上方的数据观察窗口(图1.4)，在左下方的模式编辑界面(图1.5)及在右侧的资金管理系统界面(图1.8或图1.9)。

市场输入界面(图1.2)：[English]

这部分共有九个键。从左到右它们分别是模拟模式键，多市场键，市场种类F/S(期货/股票)键，主市场符号输入键，OK键和第二、第三、第四及第五市场符号输入键(有时也称为第一、第二、第三及第四辅助市场)。期货市场的符号是由1到2个字母组成，股票市场的股票符号是由1至6个字母组成。点击OK键将把主市场的历史数据图输入到数据观察窗口。市场种类键用于控制其两侧键的意义，即主市场和多市场键。如果主市场符号显示的是C，当市场种类键是F时，它表示是玉米期货；但当市场种类键是S时，它表示是美国花旗集团。另一方面，当模拟模式键为M时，市场种类键还决定点击多市场键时弹出那个窗口。图1.3.1和图1.3.2分别显示了其两种不同的情况，关于它们我们以后再讨论。第二、第三、第四及第五市场的符号输入方式是：f/s+name，我们把这种符号输入方式记为“fsname”，f对应于期货，s对应于股票。这样我们就可以把期货和股票放在一起来做模拟，如果我们认为它们中有某些关系的话。对fsname记号而言，FC是玉米期货，SC是美国花旗集团。模拟模式键有两个选择，S-表示单市场模拟，M-表示多市场模拟。

 

当模拟模式键为M时，点击多市场键时，将以市场种类键是股票还是期货而分别弹出图1.3.1或图1.3.2。只有那些选择方框被选择了的市场才参与下一次的模拟操作。 All键选择所有的选择方框，None键放弃所有的选择。Save键先存后退出，Close键不存直接退出。用户可以从表格中删除符号或加入新符号代替旧符号。

模式文件/资金管理规则文件编辑界面：[English]

这部分中一共有十个键。最大的文字区用于输入模式文件或其有关的资金管理规则文件。在此使用者可以用MATLAB及FPW的任何函数及功能，除必须遵从MATLAB的语言结构外，还必须用分号“；”结束每一命令。如同很多别的地方一样，这里可用的变量有o - 日开盘价, h - 日最高价, l - 日最低价, c - 日收盘价及v - 日交易量等主市场的日数据数列，并用i代表数列的当前日期指针。在这个文字区之上 是文件的管理系统。从左到右，它们是删除键，文件打开键，模式/规则(P/R)键，文件名输入键，文件名指数键，文件名种类(B/S)键和存盘键。当存盘时，如重名系统将自动覆盖老文件，并不发出警告。

 

存盘Save键的右边是Run键和模拟观察键M。点击Run键则开始模拟由当前模式文件和资金管理规则描述的交易模型。点击观察键，将弹出一个观察视窗以观察模拟操作的实际进度，因为很多模拟操作可能会花费长达数分钟甚至数小时，这个观察视窗大约每10-20秒更新一次，它可以很大程度上缓解用户由於“无法控制服务器”而引起的急躁不安。图1.6显示的是一个股票多市场模拟时的观察视窗。其左上角显示着程序正在对那个股票进行模拟，还标出它在市场数列中的行和列的坐标，其行是从下往上数，其列是从左往右数。市场模拟的顺序也是从左下角向右上角一行一行地进行。显示的百分比数是对当前市场模拟的进度。另外点击打开及删除文件键将自动跳出一列有可选文件名的窗口供用户选择，如例图1.7所示。所有文件都是存在服务器里而非本地PC。

 

存在服务器里的P/R文件名是有严格规则的。对模式(P)文件格式为：文件名种类B/S (用户看不到) + 文件名(任意长，但不能有空格) + 文件名指数；对Any资金管理规则(R)文件为：文件名(四字母字符串，格式为ee/ss/oo + 用户定义的两字母字符串) + B/S + 文件名指数(对退出规则是7或8，对止损或目标规则是8或9)。点击管理规则入场中的Any键可以把Any管理规则R文件分别插入为当前Any1或Any2管理规则，当然也可以把上面的文件名四字母字符串输入为STOP，EXIT或OBJE而直接把该管理规则R文件存为当前Any1或Any2资金管理规则。资金管理规则(R)文件的文件名指数就是它们分别所在其分类组中的实际从上至下数的物理位置。看完后面关于Any资金管理规则的介绍，就会容易理解这些了。

 

在很多情况中，程序需要反复计算某些相同东西，例如各种指标。如果把这些集中起来一次处理，速度就会快很多。因此我们引进了一个特殊的全局变量MiMi，在模拟开始前先对一些东西一次性批处理，并把处理的结果交给MiMi，模拟开始后让其再传给主模式程序。为了区分开预处理部分和主模式程序部分，在模式建立文件中引进一个[/MiMi]分界记号，它之前是预处理部分，之后是主模式程序部分。如图1.5所示，它做的是一个之字信号模拟，MiMi带有信号的方向，价格及地址。重要的是我们还要记住：[/MiMi]分界记号和预处理部分只是一个选择项，可有可无，不一定必须要用要有。

模式文件的内容是固定的。它总是以相同的表达式‘y = [0 x1 i x2 x3 x4 x5 x6];’开始，其中x1至x6是具体的参数值。这里的y和i必须小写。 y是模式文件的输出，它是一个一行七列的数组，其各分量的意义是：

y(1) = 信号方向：“1”是买信号，“0”是中性即没信号，“-1”是卖信号。

y(2) = 信号入场价，即信号的实际触发价；(x1)。

y(3) = i，因i总是当前日期地址的指针，y(3)则是信号的入场地址。

y(4) = 信号触发的时间类别：1是指开盘时，2是指日间，3是指收盘时；(x2)。

y(5) = 信号触发当日模拟时，是否加入相应的资金管理规则：1是，0否；(x3)。

y(6) = 模拟时用什么数据库：1是日间数据库，0是日数据库； (x4)。

y(7) = 信号触发的方式：1是顺势，-1是逆势；(x5)。

y(8) = 过夜信号叠加是否允许：1 - 允许，其它任何数 - 不允许；系统默认值为1,既允许(x6)。如果y(8)值不为1时,其约整绝对值意为前一信号结束round(abs(y(8)))天(线,bar)后,下一个信号才可以开始。当无过夜叠加信号预期情况时,为使用户定义的特殊ESO能与其它系统提供的ESO可一起共同工作,通常要将此值设置为0,-1或其它非1数值。特殊情况: 当值设为0时,如果一个信号因止赢或止损被提前退出,系统这时候将自动按'-1'处理,既允许接受新信号的时间推迟一天,因否则将无法实际操作.

 

整个模式文件的目的就是为了回答上述七或八个分量。当y(1)不等於0时，所有其它几个分量则必须得到正确的答复。y(7)对於日数据库模拟没有意义，赋值1或-1都一样。当y(5)和y(6)都是1时，除了y(4)外，我们还必须找到信号触发的具体时间，并把其值赋给全局变量‘wstime’，而把信号触发时前一线的时间赋值给另一全局变量‘fdctime’。当进行模拟时，每一天都要对主模式文件做检测，看是否有新的信号触发，但[/MiMi]前的程序码只执行一次。

 

当编辑用户自己定义的资金管理规则文件(称为Any-ESO文件)时，与编辑模式文件有相同的步骤，但对其文件名有更严格的要求及规定(见上)。另一方面，这时函数的输出是一个1x2数列(而不再是个1x7数列)，它的分量是：y(1)是退出、止损或目标价格；y(2)是它的激发时间种类，1是开盘时，2是日间任何时候，3是收盘时。程序为每一Any-ESO都提供有两个可编辑的文字输入键，以方便用户输入参数；它们的全局变量名是(从左到右)：

 

Exit  -      Any1:     new301,  new302;                 Any2:     zhu722,   zhu723

Stop -     Any1:     new303,  new304;                 Any2:     zhu732,   zhu733

Obje. -    Any1:     new305,  new306;                 Any2:     zhu742,   zhu743

 

程序自动把相应的字符串赋值给以上相应的变量。例如在图1.8中，这时zhu722='45'，new303=' '；因此熟悉一些MATLAB字符操作函数在这里是很有帮助的，如<str2num>。

 

为了简化因而能更好地利用Any-ESO 功能, 在最新版本中,  我们在本页面的最右边引入了三大块可修改的文字输入区. 用户可以直接在那里输入任何Any-ESO 编码. 它们同主模式码一样, 会一直明显地直接显示在那里. 对它们编码时的条件或限制完全和编码主模式时一样, 这样一来可以把原来的Any-ESO功能扩展到无限. 可以和Any-ESO功能同时混合使用. 在某一文字编辑区内任何地方,只要输入Not assigned for this pattern 或者 Not available 就表示不用新功能的这一部分.

 

不论是编辑模式文件还是管理规则文件，能用的变量都是相同的，它们是数十上百个全局变量，用户必须严格按其预定含义使用，关于变量更多信息请参看变量一节。做模型最常用到的函数应该是WSLOGIC或WSTKR等.请参看子函数一节熟悉更多的可用变量。

退场(退出，Exit)选择：[English]

一共有八种退出方式，可同时选择一种以上的方式。目的是当一个信号即没有被止损也没达到目标时，为它 设一种其它的信号结束方式。从上至下，它们分别是：

 

第1行：时间到退出 - 在第y天(信号触发当天记为第零天)的什么时间或x时分线后退出。在这里一共有四个互动输入/选择键，当第一选择键显示T时，表明x(大於零的正整数)时分线结束时退出，它和后面的两个键比较后选择较早的。当第一选择键显示D时，表明在第y天的开盘/收盘(由第二点击键决定)时退出；如果是在收盘时退出，更进一步程序可以用x(这时可能不再是整数)表示几点“认作”或“假定为”结束，从而达到在一天中的任意时刻退出；当x显示20.00，其指的是市场正常收盘时间，而非是晚上八点。在图1.9中它具体表示为在信号触发日的下一交易日的开盘时退出。

 

第2行：连续式退出约定1 - 当连续保持x天(信号触发当天记为第一天)处於赢的情况下(与打平价相比))，收盘时退出。

 

第3行：连续式退出约定2 - 当保持x天(信号触发当天记为第一天)处於连续赢/输的情况下(与前一日收盘价相比))，收盘时退出。x>0对应赢，x<0对应于输。

 

第4行：累计赢退出 - 信号触发x天后(信号触发当天记为第一天)，当累计有y(<=x)天开盘至开盘/收盘至收盘(由第一点击键决定)赢，信号则在这个开盘/收盘时退出。

 

第5行：打平退出 - 信号触发x天后(信号触发当天记为第一天)，当累计有y(<=x)天其开盘/收盘价(由第一点击键决定)好于打平价，信号则在这个开盘/收盘时退出。

 

第6行：之字退出 - 用于之字形买卖模拟，其信号由MiMi触发。

 

第7行：Any1 - 用户自己定义的退出管理规则，将在本节的最后部分讨论。

 

第8行：Any2 - 用户自己定义的退出管理规则，将在本节的最后部分讨论。

 

 

止损(Stop)选择：[English]

一共有九种止损方式，可同时选择一种以上的方式。目的是当一个信号没按设计的方向走动时，为了不至於让一个信号损失太多而止损。从上至下，它们分别是：

 

第1行：美元止损 - 简单的说就是输了多少钱后必须退出。在这的选择键被称为TVD键，其有4个选择项：T(最小差价)，V(波动度),C(进价)及$(美元)。这个键也同时控制其它几个止损及目标键的功能。根据TVD键的显示，止损量可分别为：x个最小差价(TVD=T)，其信号前5日平均交易范围的x倍(TVD=V，x可以大於1), 进价的x个百分点(TVD=C)或x百美元(TVD=$)。

 

第2行： 枢纽点止损 - 信号触发之前的枢纽点延伸x个最小价差为止损 。第一选择键用于选择用几级(1-9)枢纽点。买信号：枢纽低点-x个最小价差；卖信号：枢纽高点+x个最小价差。

 

第3行： 跟踪止损 - 在这里有两个选择键和一个数字输入键。总的说就是保持止损价与日线或日间线的高/低/结束价(由第二点击选择键决定)x距离，其x的计算单位是由TVD键(见美元止损)决定的。但有两个例外：1，TVD=V或C时，x的计算单位也是最小差价；2，当第二点击选择键显示“$”时，止损价与前一天的结束价的距离总是x百美元。当第一点击选择键显示“0”时，表明跟踪止损要等信号好于打平价以后才开始用；当其显示“1”时则从信号一开始就实施。在信号日, 跟踪止损总是以进价为参照的, 与H,L,C显示无关.

 

第4行： 时间打平止损 - x(要求>0，信号触发当天记为0)天后，如果前一交易日的收盘价好于打平价，则止损价设为打平价。

 

第5行：打平保护止损 - 当信号在前一线已赢至少x个最小差价(TVD=T,C或V)或x百美元(TVD=$)后(这里赢的定义是前一线的结束价与打平价相比)，则把止损价从当前线设为打平加赢y个最小差价(TVD=T,C或V)或y百美元(TVD=$)。这里并不要求 x>=y，但是如果y<x，信号则被立刻“止损”。

 

第6行：图标止损 - 用信号触发前x(>0)天的最高价/最低价/结束价(由第一点击选择键决定)中的最高价(卖信号)或最低价(买信号)止损。

 

第7行：V-T止损 - 止损价定为x(可>100)百分点其信号前5日平均交易范围(即mean(h(enw-5:enw-1)-l(enw-5:enw-1))或vo(h,l,enw-1,5))，但止损不小于y个市场最小允许波动价。

 

第8行：Any1 - 用户自己定义的止损管理规则，将在本节的最后部分讨论。

 

第9行：Any2 - 用户自己定义的止损管理规则，将在本节的最后部分讨论。

 

 

目标(Obje.)选择：[English]

共有九种目标方式，可同时选择一种以上。目的是当一个信号按设计的方向走动到一定程度时，为避免它们再流失而及时收回或部分收回所赢。除特别指出外，信号达到目标时要求至少能打平。从上至下，它们分别是：

 

第1行：美元目标 - 除把输赢反一下外，其意义与美元止损相同。但这里有两个值，其中第二个值要比第一值大。程序将按入场方式第四行中输入的参数按比例分两次逐渐达到不同的目标。

 

第2行：报答目标 - 目标距离是x倍最大初始止损距离。

 

第3行：疯狂日目标 - 如果今天的交易范围是过去5天平均交易范围的x倍，且结束在今天交易范围的y百分比之上(买信号)或之下(卖信号)(对於今天的低y=0，高为y=100)，则在今天收盘时结束此信号。

 

第4行：枢纽点目标1 - 信号触发之前的枢纽点延伸x个最小价差为目标 。第一选择键用于选择用几级(1-9)枢纽点。买信号：枢纽高点+x个最小价差；卖信号：枢纽低点-x个最小价差。

 

第5行：枢纽点目标2 - 当结束价累计x天在信号触发前的枢纽高/低点(对买/卖信号)之上/下时，信号则在今天结束时退出。

 

第6行：图标目标 - 用信号触发前x天的最高价/最低价/结束价(由第一点击选择键决定)中的最高价(买信号)或最低价(卖信号)作为目标价。要求信号至少要赢5倍的sl5(注：赢1倍的sl5是打平)。

 

第7行：V-T目标 - 目标价定为x(可>100)百分点其信号前5日平均交易范围(即mean(h(enw-5:enw-1)-l(enw-5:enw-1))或vo(h,l,enw-1,5))，但不小于y个市场最小允许波动价。

 

第8行：Any1 - 用户自己定义的目标管理规则，将在本节的最后部分讨论。

 

第9行：Any2 - 用户自己定义的目标管理规则，将在本节的最后部分讨论。

 

所有上述ESO们各自相互独立。对信号的约束条件只能是越来越紧。当选用了多个ESO时，程序选择最紧(即最容易实现的)或最先实现的那个。除跟踪止损外，所有ESO的值一计算出后就固定不变了。另外对所有信号，参数都将是一样的，除非在模式主程序中通过直接改变其全局变量来改变它们。把鼠标置于参数键上方，其全局变量名则显示在页面的状态显示区或该键的提示信息里。程序做这样的选择意在避免特意得出超级拟合结果。

 

 

入场(进场,Enter)选择：[English]

这部分控制着模拟如何进行及对哪一部分进行模拟。除标题行外，它有八行互动键。

 

标题行：这里的选择键控制着目前编辑的文件或进行中的模拟是对买信号还是卖信号模型。注：并不严格要求信号触发时买(卖)模式必须赋值其y(1)一定是1(-1)，否则之字模型则没法做了。其主要作用是文件分类。

 

第1行： 不论P/R编辑区正在干什么，点击‘Disp’键用于显示当前的B/S模式文件。接下来的两个数字表示模拟范围。第一个表示从离最早数据的第几天后开始模拟，第二个表示到离最近数据的第几天前结束模拟。

 

第2行： 由於上面的模拟范围参数不太直观，FT-Link键直接把它们与数据观察窗口中的FT日期相连，以帮助设定模拟范围。另外其前面的那个Review键用于打开存在服务器里的B/S模式文件。这将首先弹出一个列有所有文件名的对话窗口，供决定打开那个文件。

 

图1.10 – 如何选择模拟范围

 

第3行：这里第一个输入键显示的是主市场最小差价所对应的美元值(每一期货合同或每1000股股票)，这个值随市场输入时自动改变，用户不必自己输入。点击键用来控制下次模拟运行时是一个全新模拟(R-NEW)或还是仅仅对上一次的模拟结果改用新的资金管理规则(R-OLD)。对后者而言，由於不需要再去确定都那些天有信号，故运行速度较快。但需注意的是：1，这个功能仅对资金管理规则不改变信号的产生的模式正确。2，其选择仅对单一市场模拟有效，多市场模拟总是做全新模拟。

 

第4行：这里x表示第一次达标时收回多少百分比的仓位。第一次达标后，如果这时止损价还低于打平价时，程序将根据右边的点击键决定是否将止损价收紧至打平价。如点击键显示是“1”则收紧，是“0”则不收紧。

 

第5行： 这里第一个输入键显示的是能整数比例分次达到目标的最小公约数，这个数是自动产生的，用户不必自己输入。第二个输入键显示的是来回交易一次的成本，即买卖差价加佣金。用户可依不同市场情况输入相应的数值，其单位是美元。模拟运行时，程序将把这个美元值依据主市场最小差价换算其成可能的最小向上接近的价格，并把这个值赋于全局变量sl5。

 

第6行： 这一行的内容只对多市场模拟时才有用。其中间的多选择菜单键列的就是之前选择的多市场组合，见图1.3.1或图1.3.2，用户在这里不是要更改它们(也不允许这么做)，而是要为每一市场配一套资金管理规则(点击右边键)或查看对每一市场都已配备了什么样的资金管理规则(点击左边键)。点击右边存放键时，将把当前资金管理规则存给当前显示的市场。点击左边文件打开键时，将把已存给当前显示市场的资金管理规则显示回为当前资金管理规则，以供查看或修改后再重新存入。当下一行右边键显示为Different时，那么做多市场模拟时不同的市场将用不同的已存好的资金管理规则；这时如果某市场没存有资金管理规则，系统将对其用当前显示的资金管理规则。但当下一行右边键显示为AllSame时，模拟运行将对所有市场都采用当前显示的资金管理规则，不论它们是否存有自己的规则。

 

第7行： 左边键是加噪声键，共有四个选择项：“AN：0”，“AN：5”，“AN：15”和“AN：30”。这“AN：”后面的数是百分比的意思，是相对于其单线交易范围的。当市场有日间数据时，先对其每线数据按比例加入噪声，然后再生成相应的日数据；如果没有日间数据时，则直接对其日数据用同样的方法加入噪声。“AN：0”表示不在历史数据上加任何噪声。加入不同的噪声后，如果模拟结果变化小於20%，则可以说模型实际交易时会是靠的住的，否则很难说。模型开发人员都知道，对过去的数据找到一些盈利的模型并不难，难的是真正实际开始交易时还赚钱。此行右边的键在解释上一行时已说明过了。

 

第8行： 这一行里的Any键用于打开用户定义的退出/止损/目标(由第二个点击键的显示内容决定)的资金管理规则并将其插入为相应的Any1或Any2(由第三个点击键的显示内容决定)。点击Any键将首先弹出一个列有所有相应规则文件名的对话窗口，供决定打开并移植那个文件。

 

 

其它及例子：[English]

在页面的最底部，点击相应的链接，使用者可以快速地转去金融模式站的任意其它允许用户使用的部分。

 

在对一个市场或一组市场做模拟之前，程序首先要先检查其数据的质量；如果数据就不存在或数据是错的，当是单一市场模拟模式(S)时，程序就停车不去进行模拟；当是多市场模拟模式(M)时，程序将把其从市场组里剔除出去。

  

模式文件例子一：

下面的程序码直接考贝自P/R编辑文字区。

 

y=[0 0 i 2 1 1 1];

if (c(i-1)<c(i-8))&(c(i-2)<c(i-3))&(h(i-2)>=c(i-3));

[z1 z2]=wslogic('c>=o+0.4*(h-l)',i,14,14,'fus'); if z1==1;

z4=z2(6); fdctime=z4; ta=wsgetdat(i,z4+1/12,20); if length(ta)>0;

if ta(1,5)~=0; z3=ta(1,1);  y(1)=1; y(2)=z3;

wstime=z4+1/12; end; end; end; end;

 

在这里主市场是S&P500指数期货(fsp)。这个模式文件是说如果一天前的收盘价低于八天前的高，两天前的收盘价低于三天前的收盘价，及两天前的高不低于三天前的收盘价；则在美东下午两点时以市场价买进fsp如果这时的fus(即美国长期债券期货)高于其今日开盘价加40%的其今日交易范围(即其今日当前最高价减去其今日当前最低价)。

 

模式文件例子二：

下面的程序码也是直接考贝自P/R编辑文字区，它的第二(或第一辅助)市场是fus。

 

y=[0 0 i 2 1 1 1]; tf=13.5; tt=15; if c(i-2)<h(i-8);

if ws2nd(sj(i,:),'c(i)>=l(i-1)')>0;if ws2(i,tf,tt,'fus',2,0)>=4/32;

sskk=wsgetdat(i,tf,tt,'fsp'); fdctime=tt; if length(sskk)>0;

sk=wsohlcv(sskk); if sk(4)>=sk(1)+1; t=wsgetdat(i,tt+1/12,20);

if length(t)>0; if t(1,5)~=0; y(1)=1; y(2)=t(1,1); wstime=tt+1/12;

end; end; end; end; end; end;

 

在这里主市场还是fsp。 它说的是如果两天前的收盘价低于八天前的高，如果在1:25pm至3:00pm期间fus至少上涨八分之一并且fsp也至少上涨一个大点，则在3:00pm时以市场价买进fsp如果这时的fus收盘价不低于其前一交易日的最低价。

 

资金管理规则文件的例子：

下面是FPW内部固有的止损规则7，即V-T止损规则。对买信号是“stopb7”，它的程序码是：

 

if length(stl)==0; stl=-1; end; 

global ent01 new401 new402;

yuan1 = fpwrn2n(new401)*vo(h,l,i-1,5)/100;

yuan2 = fpwrn2n(new402)*tickstep;

yuan=wd(max([yuan1 yuan2]),tickstep,-1);

                                y=[max([0 enp-yuan stl]) 2];

 

对卖信号是“stops7”，它的程序码是：

 

if length(stl)==0; sts=1000000;end;

global new401 ent01 new402;

yuan1 = fpwrn2n(new401)*vo(h,l,i-1,5)/100;

yuan2 = fpwrn2n(new402)*tickstep;

yuan=wd(max([yuan1 yuan2]),tickstep,1);

                                y=[min([enp+yuan sts]) 2];

 

其中<fpwrn2n>就是MATLAB函数<str2num>加一些必要的防错措施。
