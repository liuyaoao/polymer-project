附录II: FPW的全局变量[English]

注：除特别指出外，当我们在手册中谈日间数据或日间股图时，我们指的都是5分钟间隔的。

FPW用了数百个全局变量。同MATLAB及FPW子函数和命令名一样，使用者必须按它们的本意使用，并且不能再为其做任何其它的定义，否则FPW将会在运行中出差错。

使用者不可使用的全局变量：

任何含有字符串‘show’, ‘ent’, ‘stop’, ‘exit’, ‘obje’, ‘fig’, ‘ning’, ‘zhu’, ‘mr’, ‘sy’, ‘ws’, ‘st’, ‘sw’, ‘view’, ‘name’,‘us’及‘sp’ 的变量，不论大小写。这些大部分是用来控制FPW的运行流程的。

使用者将要使用的专用变量或全局变量：

        关于数据说明: 在 FPW平台上同时提供七套现成数据及一些预先处理过的常用指标. 它们分别是日, 周数据及 5, 10, 15, 30 和 60 分钟时分数据.

        有关日线系列的数据*:

            i - 主市场的当前日地址指针

            DDwj - 每日对应的周线地址, 故DDwj(i)就是当前周线地址.

    stock - 主市场的日线价格矩阵，格式为[o h l c v]，即[开始价  最高价  最低价  结束价  交易量]

    datem, sj - 这两个是完全一样的, 主市场的日期矩阵，格式为[m d y]，即[月  日  年]

    o, h, l, c, v - 分别为主市场的日开始价、最高价、最低价、结束价及交易量数列
            dsa3 dsa5 dsa10 dsa15 dsa20 dsa25 dsa50 dsa100 dsa200 dsa250  - 3, 5, 10, 15, 20, 25, 50, 100, 200 及 250 天的 简单均线数列, 在各参数后加'm'表示其动量(momentum).
            dea3 dea5 dea10 dea15 dea20 dea25 dea50 dea100 dea200 dea250  - 3, 5, 10, 15, 20, 25, 50, 100, 200 及 250  天的 指数均线数列, 在各参数后加'm'表示其动量(momentum).
            dvo5 dvo10 - 5 或 10 天的日均振幅数列., 在各参数后加'm'表示其动量(momentum).

    drs7 drs14 drs28 drs100 drs200 - 7, 14, 28, 100 或 200 天的传统 (用SMA)RSI数列, 在各参数后加'm'表示其动量(momentum).

    dre7 dre14 dre28 dre100 dre200 - 7, 14, 28, 100 或200 天的JWW式 (用变形EMA)RSI数列, 在各参数后加'm'表示其动量(momentum).

    dmdhm - 参数为12-26-9的标准 日MACD柱数列, 在参数后加'm'表示其动量(momentum).

    dvix - vix 指数的收盘值. 在参数后加'm'表示其动量(momentum).

    dmaxh20 dmaxh50 dmaxh110 dmaxh220 - 过去 20, 50, 110 或 220 烛间的高.

    dminl20 dminl50 dminl110 dminl220 - 过去 20, 50, 110 或 220 烛间的低.

    dbb20 - 标准日 BB线数列, 在参数后加'm'表示其带宽动量(momentum).

    dadx dadxma3  - 参数为14-1烛的日平均方向指数及其3烛均值,  在各参数后加'm'表示其动量(momentum).

    dpsar dpsard dpsardma3 - 参数为0.02-0.2-0.01的日PSAR指数及其方向和方向的3烛均值.

    dcci - 参数为14烛的日CCI指数,  在后加'm'表示其动量(momentum).

    dwpr dwpr7 dwpr14 dwpr28 dwpr50 dwpr110 dwpr220  - 参数为14, 7, 14, 28, 50 ,110及220烛的日William %R 指数,  在它们后加'm'表示其动量(momentum).

    dpc1 dpc2 dpc3 dpc2ma3 - 参数为20烛的日PC指数的下,中及上沿值, dpc2ma3为中值的3烛均值. 在中值及其均值后加'm'表示其动量(momentum).

    dmfi - 参数为22烛的日MFI指数,  在后加'm'表示其动量(momentum).

    dtsi - 参数为25-13烛的日TSI指数,  在后加'm'表示其动量(momentum).

    dobv dobv3h/l dobv5h/l dobv10h/l dobv20h/l - 日OBV值及其过去3, 5, 10和20烛的高或低值.

    dvpt dvpt3h/l dvpt5h/l dvpt10h/l dvpt20h/l - 日VPT值及其过去3, 5, 10和20烛的高或低值.

有关周线系列的数据*:

     iw - 主市场的当前周地址指针, ==DDwj(i)

     DDw - 主市场的周线价格矩阵，格式为[m d y o h l c v]，即[月 日 年 开始价  最高价  最低价  结束价  交易量]

     wop whi wlo wcl wvo - 分别为主市场的周 开始价、最高价、最低价、结束价及交易量数列
             wsa3 wsa5 wsa10 wsa15 wsa20 wsa25 wsa50 - 3, 5, 10, 15, 20, 25, 50 周线的简单均线数列, 在各参数后加'm'表示其动量(momentum).
             wea3 wea5 wea10 wea15 wea20 wea25 wea50 - 3, 5, 10, 15, 20, 25, 50 周线的指数均线数列, 在各参数后加'm'表示其动量(momentum).
             wvo5 wvo10 - 5及 10 周线平均振幅数列, 在各参数后加'm'表示其动量(momentum).

     wrs7 wrs14 wrs28 - 7, 14 或 28 周线的传统 (用SMA)RSI数列, 在各参数后加'm'表示其动量(momentum).

     wre7 wre14 wre28 - 7, 14 或28 周线的JWW式 (用变形EMA)RSI数列, 在各参数后加'm'表示其动量(momentum).

     wmdhm - 参数为12-26-9的标准 周MACD柱数列, 在参数后加'm'表示其动量(momentum).

     wbb20 - 标准周 BB线数列, 在参数后加'm'表示其带宽动量(momentum).

TD350Switch - 1:  是否要FPW平台直接提供时分数据(t5 t10 t15 t30 and t60) ? 1 - 要提供(默认值),  0 - 不需要.

    t5H3, t5L3, t5H6, t5L6, t5H12, t5L12, t5H24, t5L24: 信号烛前 3, 4-6, 7-12间 及 13-24 烛间的(5分钟烛)高和低. 仅用于PatternWorkshop Pro 页面的Any 功能.

    j52 - 当前日最后一个时分线在变量 TD350, t5o, t5h, t5l, t5c,t5v等中的地址指针

    T10i - 每一 5 分钟线 (t5)在10分钟线(t10)数列中 对应的地址数列, 故j101=T10i(j51), 见下.

    T60i - 每一 5 分钟线 (t5)在60分钟线(t60)数列中 对应的地址数列
            t5sa7 t5sa14 t5sa30 t5sa50 t5sa100 t5sa200 - 7, 14, 30, 50, 100, 200 的五分钟线的简单平均值数列, 在各参数后加'm'表示其动量(momentum).
            t5ea7 t5ea14 t5ea30 t5ea50 t5ea100 t5ea200 - 7, 14, 30, 50, 100, 200 的五分钟线的指数平均值数列, 在各参数后加'm'表示其动量(momentum).
            t5vo5 t5vo10 - 5及 10 线的5分钟数据(t5)的平均振幅数列, 在各参数后加'm'表示其动量(momentum).

    t5rs7 t5rs14 - 7或14线5分钟数据(t5)的传统 (用SMA)RSI数列, 在各参数后加'm'表示其动量(momentum).

    t5wpr7 t5wpr14 t5wpr28 - 7, 14或28线5分钟数据(t5)的WPR值[-100 0], 在各参数后加'm'表示其动量(momentum).

    t5pc1 t5pc2 t5pc3 t5pc2ma3 - 参数为20烛的5分钟数据(t5)的PC指数的下,中及上沿值, t5pc2ma3为中值的3烛均值. 在中值及其均值后加'm'表示其动量(momentum).

    t5re7 t5re14 - 7或14线5分钟数据(t5)的JWW式 (用变形EMA)RSI数列, 在各参数后加'm'表示其动量(momentum).

    t5mdhm -  参数为12-26-9的标准5分钟数据(t5)的MACD柱数列, 在参数后加'm'表示其动量(momentum).

    t5bb20 - 参数为xxx的标准5分钟数据(t5)的 BB线数列, 在参数后加'm'表示其带宽动量(momentum).

    t5cci - 参数为14的标准5分钟数据(t5)的 cci指标数列, 在参数后加'm'表示其带宽动量(momentum).

    t5mfi - 参数为22的标准5分钟数据(t5)的 mfi指标数列, 在参数后加'm'表示其带宽动量(momentum).

    t5psard t5psardma3 - 参数为0.02-0.2-0.01的标准5分钟数据(t5)的PSAR方向和方向的3烛均值.

    t60wpr7 t60wpr14 t60wpr28 - 7, 14或28线60分钟数据(t60)的WPR值[-100 0], 在各参数后加'm'表示其动量(momentum).

    t60pc1 t60pc2 t60pc3 t60pc2ma3 - 参数为20烛的60分钟数据(t60)的PC指数的下,中及上沿值, t60pc2ma3为中值的3烛均值. 在中值及其均值后加'm'表示其动量(momentum).

    t60re7 t60re14 - 7或14线60分钟数据(t60)的JWW式 (用变形EMA)RSI数列, 在各参数后加'm'表示其动量(momentum).

    t60mdhm - 参数为12-26-9的标准60分钟数据(t60)的MACD柱数列, 在参数后加'm'表示其动量(momentum).

    t60bb20 - 参数为xxx的标准60分钟数据(t60)的 BB线数列, 在参数后加'm'表示其带宽动量(momentum).

    t60cci - 参数为14的标准60分钟数据(t60)的 cci指标数列, 在参数后加'm'表示其带宽动量(momentum).

    t60mfi - 参数为22的标准60分钟数据(t60)的 mfi指标数列, 在参数后加'm'表示其带宽动量(momentum).

    t60vri10 t60vri22 - 参数为10/22的标准60分钟数据(t60)的 vri指标数列.

    t60adx - 参数为14的标准60分钟数据(t60)的 adx指标数列, 在参数后加'm'表示其带宽动量(momentum).

    t60psard t60psardma3 - 参数为0.02-0.2-0.01的标准60分钟数据(t60)的PSAR方向和方向的3烛均值.

*: 在上述变量中, 每一个都可在模式编辑框中使用, 但只有黑体部分可以在输出页面中的Any框中使用, 对应着信号前一条线的相关数据.

enp - 当前信号的入场价

enw - 当前信号的入场地址指针

endi - 当前交易方向. 1: 多, -1: 空, 0: 无仓.

datem, sj - 这两个是完全一样的, 主市场的日期矩阵，格式为[m d y]，即[月  日  年]

name - 主市场的符号

mname - 主市场的种类, ‘S’为股市，‘F’为期货

sl5 - 买卖差加佣金，以主市场的价格单位表述

riskd - 信号的启始最大风险，即启始止损价与入场价之差的绝对值

tickstep - 主市场的最小价格变化量

intra - 模拟时是否进行日间检测指标，1:是； 0:否

stk, stk3, stk4, stk5 - 分别为第二、第三、第四及第五市场的价格矩阵，格式为[o h l c v]

dam,dam3,dam4,dam5 - 分别为第二、第三、第四及第五市场的日期矩阵，格式为[m d y]

ycp - 主市场相对于当前地址指针的前一日的关门价

trendv - (Workshop专用) 信号触发种类指针，1：顺势， -1：逆势

fdctime - (Workshop专用) 做日间模拟时，信号第一天开始做SEO检测的时间, fdctime>=wstime(见下), 如果fdctime>wstime,这意味着它们之间的数据不做SEO检查.

wstime - (Workshop专用) 信号的触发时间

MyOT - (Workshop专用)  主市场当日开盘时间

MyCT - (Workshop专用) 主市场当日收盘时间

lotshis - 当前模拟系统的信号累计交易量数列，同其它参量一起控制交易频率及信号的渐入渐出

wspt - 部分净利提取指标，1：已做部分净利提取，0：还没做

MiMi,MiMi2,MiMi3,MiMi4 - 都用于通常的之字模拟，即---买-空仓或卖-再空仓或买---反复做的模拟；MiMi此时格式为：[方向 进场价 进场时间 其它参量]. 另外,四个都可以用于由预处理程序(由[/MiMi]符号分开)向主模式建立程序输入处理后的变量。它们的初始值都是空的,既[]. 下面是之字模拟的一般主程序，和第6退出一起使用。
y=[0 c(i) i 3 0 0 1];
if (MiMi(i,1)==1)&(lotshis(i)~=1)&(MiMi(i-1,1)~=1);
         y(1:2)=MiMi(i,1:2);
elseif (MiMi(i,1)==-1)&(lotshis(i)~=-1)&(MiMi(i-1,1)~=-1);
         y(1:2)=MiMi(i,1:2);
end;

lcps - 用户待定全局变量，便於程序间数据的传输

ltpd - 当前信号的信息1x6数组：[入场价*方向, 触发日地址, 触发时间(小时), 出场价, 出场日地址, 出场(小时)]，方向定义是： 1：买，-1： 卖。模拟开始时的初始值设为[0 0 0 0 0 0]，另外从tradeh可以获取更多的关于信号的信息。象如果ltpd(2)>0; tradeh(length(tradeh(:,1)),8)将给出信号进场是星期几。

tradeh或TH - 交易信号历史数列，每一行代表一个信号，按日期及时间排列。它包含所有关于该信号的必要信息，共有九列：[入场价，方向，入场地址，出场价，出场地址，信号美元输赢，出场方式代码(止损,退出或达标)，入场时为星期几(1-5)，入场时间(0-24)]。模拟开始时设为：[0 0 0 0 0 0 0 0 0]。出场方式代码：[11  12  13] - [开盘 日间 收盘]止损，[21  22  23] - [开盘 日间 收盘]退出，[31  32  33] - [开盘 日间 收盘]达标。

CPS - 用户给每个交易加的参数, 用于Any框中做特设统计. 如在模型建立处写上"CPS=h(i-1);" 在输出结果的ANY 处, "h(i-1)>=l(i-2)" 或 "CPS(ii,1)>=l(i-2)" 将给出同样结果. 当想用CPS同时传递多个参数时, 只要把它们都写在同一行就可以了. 如 "CPS=[h(i-1) c(i-1) l(i-1)];", 则 CPS(ii,2) 相当于c(i-1), 其余类推. 这个参数仅在模型开发时有用, 其它地方没任何用处. 所以模型开发好后, 应该把CPS从模式描述中去除.

TPH - 信号在账簿的位置历史，是个两列矩阵: [绝对位置数 代数位置数]. 只有当输出是来自PM的"Run This Combination"时才会有这个参数, 其它时候为两列全"1"矩阵.

TD - 信号日期历史，它的定义为TD=datem(TH(:,3),:)。

TM - 信号的模型信息, 为两列矩阵,定义为以前仅程序内使用的TH(:,12:13). 第一列是合成时各模型加入的序列号, 从1到模型合成数,最高可高达99. 第二列是每个信号存在服务器中的文件号(也是1到99), 当没有这个数据或不是来自 Portfolio Manager 的输出时,此列结果设为０．

TX - 日间模拟输出时,信号的退出时间。仅在输出页面中的Any框中使用. 其它地方用TStop, TObje 或 TExit. 这里的TX是这些变量的综合结果. 它是一个1x1 变量, 用时不需要象用TH, TD, TM等要注明分量指标, 如 TX>12 [而不是 'TX(ii,:)>12'] 表示在下午出场的交易.

ltdss - 当前交易日模拟状态参数, 初始值为 [1 0 0 0 0 0 0 0]. 利用它可以完成多同一日内多信号的模拟. 当把其第一个元素的值设为'0'时, 程序将继续对当前日做日内模拟, 起始时间点可由模式输入部分控制. 在那里人们还可以用ltdss的其它元素传递更多的模拟信息。tot - 1x4 数列. 对应于第一退出设定的四个参数 [1('T')/2('D') x y 1('O')/3('C')] ，仅在模式建立程序中用.它比下面介绍的totc具有更高的优先权.

MyNexti - 用之前首先需要做 global 声明. 它最常用于几个客户化的交易管理和模式建立的编码中. 这个参数决定程序在搜寻下一个信号时从哪开始. 如果没用用, 它的值为零, 则程序按正常次序从上一个信号触发日的第二天开始搜寻下一个信号. 它可用于快速跳过一段时期, 加速模拟速度. 例如: "MyNexti=i+10;" 它告诉程序直接从当前日"i"蹦到10天后开始寻找下一个信号.

LABAC - [每一信号在同一天被触发的绝对次序号, 每一信号在买(+1)卖(-1)相抵后该信号在一天内的累计代数值, 过夜信号累计代数值], 是一个三列的矩阵, 和TD及TH总是有相同行,并一一对应. 用在上ANY里. 例如, abs(LABAC(ii,2))<=4 任何时刻都将限制4个净仓位,. abs(LABAC(ii,3))<=2 将过夜仓位限制为2个. 而 LABAC(ii,1)==1 将只接受每天的第一个信号 (如果多于一个时).

VCRAF -  用在上ANY里. 这个变量大小写不灵敏, 都可以. 它用于通过交易量控制风险. 如 VCRAF(ii)<=c(i-1)/100, 程序将按 vo(h,l,i-1,5) [vo5] 以 c(i-1)/100 (既 1%) 为界, 高处交易部分将同比例缩小交易量.

STATOUT -  用在上ANY里.  这个变量大小写不灵敏, 都可以. 它的格式是 "Statout : statement". 程序将把statement的统计结果输出到 Status窗口中. 如 StatOut: sum(abs(lotshis'))==0 将告诉我们有多少天一个信号都没有.

brkl, brks - 买、卖信号打平价(考虑到sl5之后)

bkei - 对当前信号而言，信号打平指标, 1：打平了,  0：还没打平

stl, sts - 买、卖信号强行止损价，在模式建立程序中输入

obl, obs - 买、卖信号强行达标价，在模式建立程序中输入

tot - 1x4 数列. 对应于第一退出设定的四个参数 [1('T')/2('D') x y 1('O')/3('C')] ，仅在模式建立程序中用.它比下面介绍的totc具有更高的优先权.不论之前状态如何,只要一用tot,程序将自动把退出选项一选择为选了.这是它不同于totc的另一个地方.

totc - SEO 参数控制结构变量,仅在模式建立程序中用. 例如 totc.zhu61='3'; 命令将把第一退出选项中的第二个数字项置为3. 通过移动鼠标到想要控制的选项上可以获得应该使用什么结构分量名. 这个变量不改变任何SEO选择状态.

exitpai, stoppai, objepai - 控制是否允许对Any-EOS输出的价格做任何修正? 0表示不可以, 1表示允许. 只需在相应的价格输出前提示"xxxxpai=0;" 或 "xxxxpai=1;" 默认状态是允许,即1. 当设为0时, 用户必须保证提供的输出价格一定是在当日范围之内, 否则程序就会出错.

TExit - 一个1x9数组, 存着每个退出机制可能发生的具体时间. 单位是小时. 没可能发生的设为100. 第一个数对应最上面的时间退出那个. 第8个对应 Exit Any2. 第9个对应Special Any-Exit. 对于Any1, Any2 及 Special Any rules, 用户需要提供相应可能退出时间. 如对special any情况 用户需把可能退出时间赋予TExit(9), 否则程序可能不会作出正确的退出选择.

TStop - 一个1x10数组, 存着每个止损机制可能发生的具体时间. 单位是小时. 没可能发生的设为100. 第一个数对应最上面的$Mg那个. 第9个对应 Stop Any2. 第10个对应Special Any-Stop. 对于Any1, Any2 及 Special Any rules, 用户需要提供相应可能止损时间. 如对special any情况 用户需把可能止损时间赋予TStop(10), 否则程序可能不会作出正确的止损选择.

TObje - 一个1x10数组, 存着每个达标机制可能发生的具体时间. 单位是小时. 没可能发生的设为100. 第一个数对应最上面的$Mg那个. 第9个对应 Obje Any2. 第10个对应Special Any-Objective. 对于Any1, Any2 及 Special Any rules, 用户需要提供相应可能达标时间. 如对special any情况 用户需把可能达标时间赋予TObje(10), 否则程序可能不会作出正确的达标选择.

PatSeed - 一个20x50的矩阵, 每一行对应PW Pattern Setup (在MarketPulse, 用户通过提供的GUI可自由修改每一元素). 它的第一行是PatWeight.

PatWeight -  模式相关计算权重, fpwpatcheck函数用.默认值是PatSeed的第一行, 但不限制, 用户可自由替换.

MyPat - 待寻找模式,默认值是PatSeed的第二行, 但不限制, 用户可自由替换.

 

PatSeed - It is a 20x50 matrix, each row is a pattern corresponding to PW Pattern Setup index (User can edit them using provided GUI located in MarketPulse). Its first row is for PatWeight.

PatWeight - The counted weight factores vector, that is used when do special pattern scan. Default is the first row of PatSeed, but user can user whatever they want.

MyPat - It is a row vector, default is the second row of PatSeed, but user can user whatever they want.

 

 

特殊变量: 下面的变量是为 'wslogic1'和 'wstkr'函数用的. 正确用法是在所用函数之前输入.

wsgp,wsgp1 - FPW函数<wslogic(1)>专用，从主模式建立程序向其输入参量.

wsgp2,wsgp3 - FPW函数<wstkr)>专用，从主模式建立程序向其输入参量.

wspc - FPW函数<wslogic(1)>专用，向 其输入是否进行检查的前提条件.只有这个条件为真(非零)时,才做进一步'logic'的检查.默认值总为真.

wspo - FPW函数<wslogic(1)>专用，向主模式建立程序输 出参量.

wspo1 - FPW函数<wstkr)>专用，向主模式建立程序输 出参量.

wsgf,wsgf1 - FPW函数<wslogic1>专用，从主模式建立程序向其输入需计算的字符串,如: [wsgf='mLzn=max([mLsa30 mLea30],[],2);'].

wsmh - FPW函数<wslogic(1)>专用，从主模式建立程序向其输入参量以控制日间数据种类.用之前必须先声明.其值必须是5,10,15,30或60分别对应于5,10,15,30或60分钟日间数据.错误输入数据或没有输入时其值为5.另外, 用户还可以用wsmh输入一个 1x2 数组告诉程序需要载入多少天 (范围: 1-60) 的日间时分数据, 程序默认值是5天. 这时它的第一个分量代表的是上面讨论的数据种类, 第二个分量责是指定载入多少天的之前数据. 例如"wsmh=[10 23];" 则指定载入23天之前数据, 每个线代表10分钟. 天数越大, 速度越慢.
