Appendix II: FPW Global Variables

Note: Tick data means intraday data or intraday bars (5 minutes bars in most cases) except specially explained.

FPW has hundreds of global variables.  Like MATLAB and FPW function and command names, one should also use them as what they are.  If one has to use them and should not name ones own variable names same as them. 

 

Global variables lists you could not use:

Any name which contains strings of ‘show’, ‘ent’, ‘stop’, ‘exit’, ‘obje’, ‘fig’, ‘ning’, ‘zhu’, ‘mr’, ‘sy’, ‘ws’, ‘st’, ‘sw’, ‘view’, ‘name’, ‘us’, 'wsgp' and ‘sp’ in either upper or lower case.  Most of these are used to control the program flow.

intra - index for checking intraday data or not during a simulation, 1: yes, 0: no

stk, stk3, stk4, stk5 - 2nd,3rd,4th and 5th markets’ price array in the format of [o h l c v] respectively

wstime - (Workshop) Signal triggered TIME for tick data simulation

MyOT - (Workshop) market open time of the current day for tick data simulation

MyCT - (Workshop) market close time of the current day for tick data simulation

wspt - profit taking index: 1: partially taken, 0: not yet

MiMi, MiMi2, MiMi3, MiMi4 - They are all used to do common zigzag simulation (long, neutral and  short back and forth) with format of [direction, EntryPrice, EntryTime, Others] for MiMi. One can also useall of them to pass parameters to the main setup program before [/MiMi] sign in the setup section. Initially, the are all empty, i.e. [].  Here is a sample for its general format for the main pattern setup program for a zigzag simulation and uses with Line 6 Exit together.
         y=[0 c(i) i 3 0 0 1];
         if (MiMi(i,1)==1)&(lotshis(i)~=1)&(MiMi(i-1,1)~=1);
                 y(1:2)=MiMi(i,1:2);
         elseif (MiMi(i,1)==-1)&(lotshis(i)~=-1)&(MiMi(i-1,1)~=-1);
                 y(1:2)=MiMi(i,1:2);
         end;

lcps - preferred user global parameter to transfer data between programs:  Its initial value is [0 0 0 0 0 0 0 0] (eight zeros).

ltpd - latest signal’s info of 1x6 array: [Entry-Price*Direction, Entry-Date-index, Entry-Time, Exit-Price, Exit-Date-index, Exit-Time], where direction definition is: 1 long, -1 short.  Its initial values before simulation began is [0 0 0 0 0 0].  To access more latest trade's information, check tradeh, e.g. if ltpd(2)>0; tradeh(length(tradeh(:,1)),8) will be the latest signal's in weekday number.

tradeh (TH) - trade history array, row number is in the order of the triggered date and time. It contains all basic information about that trade, Each row, i.e. for each trade, contains the following nine columns format information:
[entry price, direction, entry place (index), exit price, exit place (index), dollar net, exit way code (stop, exit or hit objective), entry day's weekday  number (1-5), entry time (in military time format)], its initial values before simulation began is [0 0 0 0 0 0 0 0 0]. Exit way code: [11  12  13] - [Open Intraday Close] Stopped Out,  [21  22  23] - [Open Intraday Close] Exit,  [31  32  33] - [Open Intraday Close] Hit Objective.

CPS - Custumerized Parameter following each trade to do Statidtics in Any cell. For example, if one writes "CPS=h(i-1);" during pattern setup, then to enter "h(i-1)>=l(i-2)" or "CPS(ii,1)>=l(i-2)" would give the same results. When using CPS to passing more than one parameters, placing them in a ROW, such as "CPS=[h(i-1) c(i-1) l(i-1)];", then CPS(ii,2) would be c(i-1) and so on. This CPS are only used at model developing stage. Once a model done, one should drop it. It will be useless in any other part.

TPH - Trade Position History 2-column matrix: [Absolute_Position  Net_Position] in the portfolio book.  It's only available after "Run This Combination" run in Portfolio Manager page, otherwise it's an all ones matrix.

TD - trade date history, its definition is TD=datem(TH(:,3),:).

TM - trade model information, two column matrix. It is the same as TH(:.12:13) which is of internal use only before. The first column is model combining order from 1 to model numbers which is combined, can be up to 99. The second one is the saved file numbers in Portfolio Manager (also 1 - 99), 0s when not available or not from Portfolio Manager.

TX - exiting time for intraday simulation. Can be used only in the output page's Any text input area. For other places, use TStop, TObje or TExit. TX is simply a combined results of these variables. This is a 1x1 variable, not like using TH, TD, TM and others, user does not  need to name its index, for example: TX>12 [NOT 'TX(ii,1)>12'] means trades exited in the afternoon.

ltdss - Latest Trading Day Simulation Status: Its initial value is [1 0 0 0 0 0 0 0]. It is used to control muti-signal intraday simulation in a day. When its first element value sets to be 0, program will do simulation again for the same day. The restarting time can be controlled by pattern setup section, where one also can use other elements of this global variable to pass more information to control the simulation.

MyNexti - Need to do global declaring first when to use. It is most used by the couple customerized money managements and pattern setup. This parameter will decide where next signal scan to begin. If not used, its value is zero, and program will always do the scan from the next day after the last signal entry date. This parameter can be used to skip a period of days. For example: "MyNexti=i+10;" It will skip 10 days (instead of the default 1) from the current day i to begin the scanning for next signal.

LABAC - [absolute triggering order in a day, current trade's net cumulated position after long (+1) and short (-1) canceling each other in a day,  cumulated over night position number] for each trade, it is a 3-column array, and always paired with TD and TH for each row. So, abs(LABAC(ii,2))<=4 will limit positions to 4 or lower anytime; abs(LABAC(ii,3))<=2 will limit overnight positions to 4 or lower; and LABAC(ii,1)==1 will only take the first trade if  having more trades in a day.

VCRAF - used in Any upper area to do risk adjustment for trading volume. like VCRAF(ii)<=c(i-1)/100, program will normalize trading volume based on vo(h,l,i-1,5) [vo5]. Any day vo5 is higher than c(i-1)/100 (i.e. 1%) will reduce its trading volume proportionally. This variable is not case sensitive.

STATOUT - used in Any upper area. Format is "Statout: plus_a_statement", it will make statistics about the statement and output to the status window, like if entering StatOut: sum(abs(lotshis'))==0, it will tell one how many days without any trade.  This variable is not case sensitive.

brkl, brks - long and short break even prices

bkei - break even index, 1: yes, 0: no

stl, sts - long and short stop out prices, entered in setup file (used in pattern editor block)

obl, obs - long and short objective prices, entered in setup file (used in pattern editor block)

tot - timeout 1x4 vector corresponding to parameters in exit-1's forms [1('T')/2('D') x y 1('O')/3('C')], only used in pattern editor block. It has higher priority than below mentioned totc. Using tot will automatically turn on the exit option 1 if it is not checked. This is another different place comparing to totc.

totc - SEO parameter control structure, only used inside pattern editor block, e.g [ totc.zhu61='3'; ] will set the second text box's contents to 3 in the first exit option. Moving mouse over where field needs to control to show what structure field name should use. This structure variable will not change any SEO choice status.

exitpai, stoppai, objepai - to control if to allow price adjustment for special any-exit, any-stop and any-objective output prices. 0 means not allowed, 1 means ok. One just needs to include "xxxxpai=0;" or "xxxxpai=1;" right before corresponding  price output. Default values for these parameters are 1. When setting any one parameter to 0, one has to be sure that its corresponding price output must be within that day's range. Otherwise program will be wrong.

TExit - It is a 1x9 vector, saved each day's possible Exit managing rules' exit times. Unit is in hour. Un-possible hit rules are set to be 100s. 1 is for the most top one, i.e. the timeout one. 8 is for Exit Any2. 9 is for the Special Any-Exit. For Any1, Any2 and Special Any rules, user needs to give the corresponding time. like for special any case, user needs to assign value to TExit(9), otherwise the value will be 100, which may not make the right exit choice.

TStop - It is a 1x10 vector, saved each day's possible Stop managing rules' exit times. Unit is in hour. Un-possible hit rules are set to be 100s. 1 is for the most top one, i.e. the $Mg one. 9 is for Stop Any2. 10 is for the Special Any-Stop. For Any1, Any2 and Special Any rules, user needs to give the corresponding time. like for special any case, user needs to assign value to TStop(10), otherwise the value will be 100, which may not make the right stop choice.

TObje - It is a 1x10 vector, saved each day's possible Objective managing rules' exit times. Unit is in hour. Un-possible hit rules are set to be 100s. 1 is for the most top one, i.e. the $Mg one. 9 is for Obje Any2. 10 is for the Special Any-Objective. For Any1, Any2 and Special Any rules, user needs to give the corresponding time. like for special any case, user needs to assign value to TObje(9), otherwise the value will be 100, which may not make the right objective choice.

PatSeed - It is a 20x50 matrix, each row is a pattern corresponding to PW Pattern Setup index (User can edit them using provided GUI located in MarketPulse). Its first row is for PatWeight.

PatWeight - The counted weight factores vector, that is used when do special pattern scan. Default is the first row of PatSeed, but user can user whatever they want.

MyPat - It is a row vector, default is the second row of PatSeed, but user can user whatever they want.

 

Special Variables: Below variables are for 'wslogic(1)' and or 'wstkr', one shoud put them right just before to use.

wsgp, wsgp1 - only used to pass parameters into <wslogic(1)> from outside.

wsgp2, wsgp3 - only used to pass parameters into <wstkr> from outside.

wspc - strings from outside, working as pre-condion for <wslogic(1)>. Only when it is TRUE (non-zero) then to check 'logic'. Its default value is always TRUE.

wspo - only used to pass out parameters from <wslogic(1)> for others to use.

wspo1 - only used to pass out parameters from <wstkr> for others to use.

wsgf, wsgf1 - only used to pass global strings into <wslogic1> from outside, such as [wsgf='mLzn=max([mLsa30 mLea30],[],2);'].

wsmh -  only used to pass parameters into <wslogic(1)> from outside, to control intraday data type, needed to declare before to use it. Its value can be only 5, 10, 15, 30 or 60 to mean 5, 10, 15, 30 or 60-minutes bar, otherwise program will set it to 5. If not declared or other number entered, the default value is 5. On the other hand, one can use a 1x2 vector wsmh to tell program how many extra days data to load.Then the first wsmh element is for the bar type as described above, the second one will be the extra days data to load in whenever to use it. The default is to load 5 days' intraday data into the program. Such as: "wsmh=[10 23];" to load 23 days earlier intraday data and each bar stands for 10 minutes. The larger of this day number, the slower the speed will be. The range one can us is 5-60.
