 PW-1. What is PW?

 

PW is a software kit to simulate the real stock and futures financial markets with only a couple of lines of code and a couple of mouse clicks.  One can simulate almost any pattern that one can imagine with full managing rules of stops, exits and objectives, scaling in or out, over one or a group of markets with same or different managing rules for each market.

 

After logging into your account, click PatternWorkshop to access PW. Its interface is shown in Figure 1.1. It consists of four major parts: the markets input interface (Figure 1.2) at the right bottom, the historical data monitoring window (Figure 1.4) at the middle of the screen, the pattern edit interface (Figure 1.5) at the left lower part of the screen, and the money managing system (Figure 1.8 and figure 1.9), which occupies the whole right side of the screen.

PW-2. Markets input interface (Shown in Figure 1.2):

 

There are nine user interactive cells in this part.  From left to right, they are the simulation mode button, the Multi-Market button, the Stock/Future button, the Primary Market input text box, the Ok button, 2nd, 3rd, 4th and 5th Market input text boxes (sometimes also referred as 1st, 2nd, 3rd and 4th secondary market).  Future names are in 1 or 2 letters while stocks in 1-6 letters.  Clicking Ok button will put the Primary Market data in the data monitoring window. The Stock/Future button only controls the buttons on its each side, i.e. the Multi-Market button and the Primary Market input text cell.  This button will force the Primary Market symbol being explained in a way of what this button shown.  For example, if letter C is typed into the Primary Market text cell, it means corn futures if Stock/Future button is ‘F’ and it would be the stock symbol for Citicorp if Stock/Future button is ‘S’. On the other hand, Stock/Future button also controls which popup window Multi-Market button will open if being clicked when the simulation mode button is on M, which means multi-market.  These two cases are respectively shown in Figure 1.3.1 and Figure 1.3.2.  We will discuss them later in the next paragraph. The 2nd, 3rd, 4th and 5th Market input cells are for the secondary markets, symbol rule is ‘f/s + name’, called fsname in this manual, for futures or stocks.  In this way, one can simulate stock markets and futures markets together if one thinks they are related to each other in certain ways. The simulation mode button controls the next simulation is a single market (S) or a multi-market (M) run. 

 

When the mode button is on M, clicking on the Multi-Market button will open the stock or futures market list editing window, shown in Figure 1.3.1 for stocks and Figure 1.3.2 for futures.  The market shown with its checkbox checked will be in the running list. The Clicks All button will choose all markets and the None button will choose no market.  The Save button is to save the chosen markets list and then exit.  The Close button is to exit without save.  One can add new future or stock symbols into the tables to replace the old ones.

PW-4. Pattern Edit Interface (Shown in Figure 1.5):

 

There are ten user interfaces within this group, as well as the P/R Editor help link.  The biggest text area is the place for the user to input patterns or new managing rules. Its code is just like any Matlab m-file. To end each line with a semicolon ‘;’ sign is a must. On top is the Pattern/Rule file managing system.  From left to right, they are the Delete button, Load button, P/R file type button, text box for file name, the Name Index button (1-9 only), the file category button (B or S) and the Save button.  For the last two buttons, Run is to begin the simulation, and the M button will open a pop-up window to monitor the simulation. It is used to indicate the status of the simulation processes because many simulations might take a long period of time, from minutes to hours.  Figure 1.6 shows a stock multi-market simulation case.  On the upper left-hand side, it shows which market is running, including its name, its row, and the column index among the lower side lists.  Counting and the simulation order are from the left lower side to the upper right side.  The percentage number in the middle of the window indicates how much data has already been processed for the current market.  The Delete and Load buttons will open an interface to let the user choose which file name to delete or to review. Sample interfaces are shown in Figure 1.7.

 

The name of the user’s P/R file saved in the server as: b/s (not visible in review, delete or load interfaces) + name (any length) + name index for Pattern files; name (four letters only in the format of: ee/ss/oo + 2 more user defined letters) + b/s + index (7/8 for exit, 8/9 for stop and objective) for Any managing rule files.  Use the Any button to load the files to Any1 or Any2 when using them or directly save them with stop, exit or obje as the file name to proper b/s (long/short) and serial numbers 7/8/9, which is where Any1 or Any2 is located in its group.

 

In many cases, the program needs to calculate the same things for each day, like various indicators.  Simulation will run fast to calculate these repeated calculations together at one time before we run the real simulation.  Therefore we introduce a global variable MiMi to do this.  Use MiMi to transfer calculated results into the main simulation.  In order to do this, the user needs to add a [/MiMi] sign inside a pattern m-file, to separate the pre-processing part from the main patterns, like the one shown in Figure 1.5, which is a called a zigzag simulation, and MiMi carries signal directions and prices.  It is important to note that the [/MiMi] section is an option.  One does not have to use it.

The contents of the Pattern files are fixed: It always begins with the same statement of ‘y = [0 x1 i x2 x3 x4 x5 x6];’.  x1 to x6 are numbers. The lower case y and i are a must. y is the output of the Pattern file. The meanings of its components are:

 

y(1) = signal direction: 1 for long; 0 for neutral, i.e. no signal; -1 for short.

y(2) = the entry price, i.e. where the signal is being triggered (x1)

y(3) = i, the system will automatically assign the current date data index value to it.

y(4) = signal being triggered in what way: 1 means at open, 2 intra-day, 3 at close. (x2)

y(5) = to check managing rules on the first day (when the signal is triggered) or not: 1 for yes, 0 for no. (x3)

y(6) = what data to simulate: 1 for intraday data, 0 for daily data. (x4)

y(7) = trend information, 1: signal is following the trend, -1 against the trend. (x5)

y(8) = if allowing OVERNIGHT position overlap? 1 - Yes, allowed. any other number - No, not allowed. System's default value is 1, i.e. allowed. (x6). If y(8) is not 1, its absolute round integer value means round(abs(y(8))) bars after previous signal CLOSED, one can begin to take new signal. Generally, one needs to set this parameter to 0,-1 or other none-1 integer in order to make his/her special ESOs can work with other platform provided ESOs together, when no position overlap at night is expected. Special case: When it is 0 and a trade is exited earlier than expected due to hit stop or objective, system will treat this case as -1, i.e new signal allowing time will postpone one day, since otherwise it is not doable in real trade.

 

The whole Pattern file is just to answer these seven or eight numbers.  When y(1) is non-zero, all others must be answered correctly.  y(7) is meaningless for daily data simulation; therefore one can enter it in either way.  When y(5) and y(6) are in 1s, one must answer more about signal time than just y(4).  One will need to find the exact time when the signal is triggered and assign it to the global variable ‘wstime’ (workshop-signal-trigger-time) and assign its previous bar time to another global variable ‘fdctime’ (first-day-check-time). During a simulation, the program will run the Pattern file for each daily bar, but things before [/MiMi] sign run only once.

 

When one edits a user-defined managing rule file, one just does the same procedures as to edit any pattern setup file discussed above but with more restricted file name rules.  On the other hand, function outputs a 1x2 array instead of a 1x7 array.  The two function outputs are y(1): price for exit, stop or objective.  y(2): when it is triggered: 1- at open, 2 - during intraday or 3 - at a close.  Each Any-ESO(Exit-Stop-Objective, see below) has two editable text boxes to input interactive parameters.  Their object handles’ global names are (from left to right):

 

Exit  -      Any1:     new301,  new302;                 Any2:     zhu722,   zhu723

Stop -     Any1:     new303,  new304;                 Any2:     zhu732,   zhu733

Obje. -    Any1:     new305,  new306;                 Any2:     zhu742,   zhu743

 

Program will automatically assign all strings entered into Any cells to above global variables correspondingly. Like examples in Figure 1.8, now zhu722='45',new303=' '; therefore to be familiar to some Matlab string functions, like <str2num>, is helpful.

 

In order to simplifying and better using Any-ESO functions, we introduced three large editable text areas on the right in our newest version. These three text areas are used to directly write Any-ESO codes. Like pattern codes, they will be shown there all the time. All limits or conditions to code them are the same as to code main pattern. By doing this, we can expand Any-ESO functions to unlimited. One can use this new added function and old Any-ESO at the same time. Entering 'Not assigned for this pattern' or Not available yet' at any place inside a text area means do not use that new rules.

 

No matter if it is a Pattern file or a managing rule file, the available variables and functions are all the same.  The variables are a group of global variables and their meanings are already pre-defined. One must use them accordingly.  These variables and functions are listed and discussed in appendix II. When developing models, the most useful functions are WSLOGIC and or WSTKR. Please refer to chapter Functions to learn more about available variables that one can use at here.

PW-5. Money Managing System:

 

There are four sections in this part, shown in Figures 1.8 and 1.9.  They are Exit, Stop, Objective and Enter (ESOE) respectively.  We will discuss them one by one.  When it has more than one text box for an ESOE, we will use x, y, z to represent their parameters from left to right. For push buttons or dropdown menus, name them the first button, the second button, and so on, from the left to the right, too.

 

Exit:

It has eight choices and one can choose more than one choice.  Its purpose is to define ways for a signal to close its position if it still has not stopped out or hit its objectives.  They are  from top to bottom:

 

Line 1. Time Out exit: How many days and what time or how many tick bars to be out.  There are four user interfaces here: the first button decides whether it is an intraday or day out. If it is an intraday out, the third and the fourth ones are as a group and to compare with the second (this should be an integer to indicate how many tick bars to exit) to choose the earlier one.  If it is a day out, the fourth one shows whether it should be out at the open or on the close.  If out on the close for a day out, the third one indicates on which day’s close (0 means the same day as the signal triggered, 1 for the next day and so on) and the number x further tells the program at what time (local military time) as if the market were closed. If x is 20.00, it means at normal market close.

 

Line 2. Consecutive out No. 1:  Whenever x days’ consecutive closes are higher or lower than the entry day’s high or low for a long or short signal respectively, then time out on the close.  x is a non-zero positive integer entered by the user in the second interface.

 

Line 3. Consecutive out No. 2:  Whenever profits are up or down consecutively for x days (counted on close-to-close), then time out on the close.  x is a non-zero integer entered by the user on the second interface. A positive number means up while negative means down.

 

Line 4. Cumulative Up out: x days after a signal is triggered (counted from entry day), whenever it cumulates y days open-to-open or close-to-close (decided by the push button) profits up, then time out on that open or close. The second interface is used to enter the parameter x, while the third for the parameter y.  The last push button is used to choose open-to-open or close-to-close.

 

Line 5. Breakeven out: x days after a signal is triggered (counted from entry day), whenever it cumulates y days open or close (decided by the push button) at better than breakeven price, then time out on that open or close.

 

Line 6.  Zigzag Exit: It is used by long-short zigzag simulation; its signals are triggered by MiMi parameter.

 

Line 7.  Any1, user defined exit rules.  Will be discussed later in this section.

 

Line 8.  Any2, user defined exit rules.  Will be discussed later in this section.

 

 

Stop:

It has nine choices and one can choose more than one choice.  Its purpose is to cut loss and close the position when a signal does not evolve according to design.  They are  from top to bottom:

 

Line 1.  Dollar stop: Fixed value stop, is based on tick (T), volatility (V), entry price (C) and dollar value ($) controlled by the TVD push button, which is also used by other stops and objectives.  The amount is entered into the second interface, which represents how many ticks, how many times of its volatility before signal day (based on last 5 days), how many percent of entry price or how many hundreds of US dollars respectively.

 

Line 2.  Pivot stop: The Latest pivot low or high minus or plus x ticks for long or short signals respectively.  The push button is used to choose what pivot level, and use the next text box to enter the parameter x.

 

Line 3.  Trailing stop: Keep stop price x (entered in the edit button) ticks or x hundreds of dollars away from the high, low or close of the last intraday or daily bar, based on the TVD button or the second push button.  When TVD is in ‘T’, ‘C’ or ‘V’, x is in ticks; but when the second push button here is in ‘$’, x unit is in hundreds of dollars away from the last bar’s close. When TVD is in ‘$’, x is in hundreds of dollars.  The first push button controls whether the trailing stop begins after breakeven (0) or from the beginning (1) of a signal. For the signal just entry day, it is always trailing from entry price, not depended on shown H, L or C.

 

Line 4.  Time to Breakeven stop: x days after a signal in the market and its previous day’s close is better than just breakeven, then set the stop price to breakeven from the beginning of this day.

 

Line 5.  Profit to Breakeven stop: After the previous day close (daily data) or last price (tick data) is x ticks (TVD in ‘T’, ‘C’ or ‘V’) or hundreds of dollars (TVD in ‘$’) in profits (at least 5 times sl5), set the stop price to breakeven plus y ticks or y hundreds of dollars in profits from the beginning of this day (daily data) or this moment (tick data).  There is no requirement for x>=y, but if y<x, then the position is immediately 'stopped' out at x profits.

 

Line 6.  Chart Stop: Stop out at the minimum or the maximum of highs, lows, closes or opens of x (>0) days period before the signal day for the long or short signal respectively.

 

Line 7.   V-T Stop: Stop price is set to be x (can be over 100) percentage of its 5 days volatility [mean(h(enw-5:enw-1)-l(enw-5:enw-1)) or vo(h,l,enw-1,5), enw is the signal day] with y ticks minimum.

 

Line 8.  Any1: user defined stop rules.  Will be discussed later in this section.

 

Line 9.  Any2: user defined stop rules.  Will be discussed later in this section.

 

 

Objective:

It also has nine choices and one can choose more than one choice.  Its purpose is to protect your profits from fading away.  They are from top to bottom:

 

Line 1.  Dollar objectives: It’s exactly the same as Dollar stop, except these are for profits and there are two parameters here instead of one for the Dollar stop.  The second one should be larger than the first one.  They are used to take profits partially according to the rules set in the Enter section.

 

Line 2.  Reward objective: x times the maximum potential lost, i.e. the initial stop value, being assigned to the global variable riskd.

 

Line 3.  Large Range Day (LRD) objective: if today’s range is at least x times wider than the last 5 days’ average and closes at above (for long) or below (for short) y percentage of today’s range, out at close if it can breakeven.

 

Line 4.  Pivot objective 1: The latest pivot high or low plus or minus x ticks for the long or short signal respectively.  The dropdown menu is used to choose what pivot level, and use the next text box to enter the parameter x.

 

Line 5.  Pivot objective 2: Out at close whenever cumulated x days’ closes are higher or lower than the latest pivot high or low for the long or short respectively.  Breakeven is a must.

 

Line 6.  Chart objective: take profits at the maximum or the minimum of highs, lows, closes or opens of x days period before the signal day for long or short signals respectively.

 

Line 7.  V-T Objective: Objective price is set to be x (can be over 100) percentage of its 5 days volatility [mean(h(enw-5:enw-1)-l(enw-5:enw-1)) or vo(h,l,enw-1,5), enw is the signal day] with y ticks minimum.

 

Line 8.  Any1: user defined objective rules. Will be discussed later in this section.

 

Line 9.  Any2: user defined objective rules. Will be discussed later in this section.

 

All above ESOs are independent of each other. They can only get tighter.  For multiple stops, objectives or exits, the program will use the nearest or earliest one.  Except for trailing stop, all ESO values are fixed once they are calculated.  They are not changed tick-by-tick or day-by-day.  On the other hand, the parameters will be the same for all signals except one coded it to change them in their setup pattern by calling their global names directly (when the mouse is over the buttons, its global name will be shown in the status bar or inside button help tips.).  The program chooses this way to avoid super curve fitting while also speeding up the simulation.

 

Enter:

This section controls what, how and where the simulation runs. It has eight lines and the title line.

Title line: The radio control button in the title line is used to choose long or short during managing rules set up.

 

Line 1:  The push button ‘Disp’ in the first line is used to show the latest saved b/s model to the pattern editor no matter what it is currently displayed.  The next two parameters in the same line are the data periods’ day numbers where the next simulation will run. The first one is where to begin and counted from day 1, while the second one is where to end and counted from the latest day.

 

Line 2:  Because the day number is not obvious, push the FT button (the second one on this line) to view them in obvious dates to help set the parameters in the 1st line. It will use the chart dates if it is in FT-Link mode (shown in Figure 1.10), or will use whatever day numbers one inputs. The first push button is used to load previously saved pattern files. It will open an interactive window to let the user choose which file to open.

 

Figure 1.10 – How to Choose simulation Periods

 

Line 3:  Tick value in dollars for the Primary Market, i.e. the studying market.  Tick value will change automatically; the user does not need to edit it. The push button is used to tell the simulation program how to run a new simulation or just change managing rules on the most recent simulation. Because it does not need to find signal days when just using new managing rules, it runs fast. This function is only for patterns in which all signals are non-related. Otherwise, signals may not be what the original pattern is looking for.  The R-New or R-Old button works for single market mode only.  For multi-market mode, it is always in R-New.

 

Line 4:  x is the percentage how much the first profit level to take.  The next push button controls whether to move the stop price to breakeven if the stop price from the stop section is still lower than breakeven after taking the level-one profits.  0 is for no and 1 for yes to move the stop price to breakeven.

 

Line 5:  The first interface is the Minimum lot numbers to run above the line’s profit taking plan with an integer lot number. It will automatically change with the above line’s profit taking plan; the user does not need to change it.  The second one is the slippage plus commission in dollars, i.e. sl5.  During a simulation, the program will round up to a market price value based on its minimum tick step, and assign it to the global variable sl5.

 

Line 6:  For Multi-Market simulation only, the dropdown menu list is the markets one chooses to run. The user could not change this list here (use the Mkt button at the bottom line to choose markets). One chooses the market here to save special managing rules to it or to review its already saved managing rules.  During the run, it will use the currently showing managing rules for the markets which have no saved managing rules.  Use the Rules button to review saved rules and the other button to save the current showing managing rules to the selected market.  When the managing rule control button in the next line is on Different, it will use each market’s saved managing rules and the current showing rules will be applied to the markets with no saved managing rules.  When it is on AllSame, the current showing rules will be used to all markets.

 

Line 7:  The first button is called Add Noise.  AN:X, randomly add noise to tick data (if any) or daily data (if no tick data for the market) by X% percent of the (intraday tick data or daily bar in no tick data available for the market) bar's range. X=0 means to add no noise. If the simulation result changed by less than 20% by adding different noises to the raw data, one can safely say that one did not over curve-fit during a modeling processing.  Otherwise, one must be very cautious about ‘very good’ results. As we all know, it is not that difficult to find a very profitable model on computer.  The second button is the one mentioned previously: the managing rule control button.

 

Line 8:  It would be very helpful to read the pattern editor section first before one wants to use this function. This line is used to load user- defined ESOs to Any1 or Any2 in the Stop, Exit or Objective sections.  The program will open an interface to let the user choose from previously saved managing rules.

From the links on the bottom line, one can quickly switch to the program’s other parts.

Pattern file example No.1

The following codes are from the P/R editor text area,

 

y=[0 0 i 2 1 1 1];

if (c(i-1)<c(i-8))&(c(i-2)<c(i-3))&(h(i-2)>=c(i-3));

[z1 z2]=wslogic('c>=o+0.4*(h-l)',i,14,14,'fus'); if z1==1;

z4=z2(6); fdctime=z4; ta=wsgetdat(i,z4+1/12,20); if length(ta)>0;

if ta(1,5)~=0; z3=ta(1,1);  y(1)=1; y(2)=z3;

wstime=z4+1/12; end; end; end; end;

 

This is a model for the S&P index futures.  The pattern is like this: If yesterday’s close price is less than the close of eight days ago, the close of two days ago is less than the close of three days ago and the high of two days ago higher or equal to the close of three days ago, and at 2pm EDT, if US long bond future trades at above its today’s open plus 40% of its up to now range, then buy the SP market at 2pm.  Here we will not print out its money managing rules because we are still trading this one now.  But we can tell you that this has been a really profitable model for the past 15 years.

 

Pattern file example No.2

The following six lines are also from the input edit cells; its first secondary market is fus (us bond futures).

 

y=[0 0 i 2 1 1 1]; tf=13.5; tt=15; if c(i-2)<h(i-8);

if ws2nd(sj(i,:),'c(i)>=l(i-1)')>0;if ws2(i,tf,tt,'fus',2,0)>=4/32;

sskk=wsgetdat(i,tf,tt,'fsp'); fdctime=tt; if length(sskk)>0;

sk=wsohlcv(sskk); if sk(4)>=sk(1)+1; t=wsgetdat(i,tt+1/12,20);

if length(t)>0; if t(1,5)~=0; y(1)=1; y(2)=t(1,1); wstime=tt+1/12;

end; end; end; end; end; end;

 

This is also a model for the S&P index futures.  If the close of two days ago is less than the high of eight days ago, if the secondary market US is up at least one eighth of a point from 1:25pm to 3:00pm and the S&P itself is up at least 1 full point at the same period, buy the SP market at 3:00pm if today’s US close is higher or equal to yesterday’s low.  Exit at 9:45am the next morning.

 

Managing Rule file examples:

Here are the FPW built-in V-T stop rules for long and short signals. For long signals - fpwstopb7:

 

if length(stl)==0; stl=-1; end; 

global ent01 new401 new402;

yuan1 = fpwrn2n(new401)*vo(h,l,i-1,5)/100;

yuan2 = fpwrn2n(new402)*tickstep;

yuan=wd(max([yuan1 yuan2]),tickstep,-1);

                                y=[max([0 enp-yuan stl]) 2];

 

For short signals - fpwstops7:

 

if length(stl)==0; sts=1000000;end;

global new401 ent01 new402;

yuan1 = fpwrn2n(new401)*vo(h,l,i-1,5)/100;

yuan2 = fpwrn2n(new402)*tickstep;

yuan=wd(max([yuan1 yuan2]),tickstep,1);

                                y=[min([enp+yuan sts]) 2];

 

Where <fpwrn2n> is just the same as Matlab function <str2num> but with some error-proof setups.
