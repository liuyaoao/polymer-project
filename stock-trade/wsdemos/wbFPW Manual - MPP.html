MarketPulse Pro & Stock Simulation

Note: Tick data means intraday data or intraday bars (5 minutes bars in most cases) except specially explained.

Pattern Setup | Parameters | Managing Rules | Simulation & Output | The Any

The only one purpose of MarketPulse Pro is to find proper patterns for MarketPulse to scan. We already see MarketPulse is very powerful and easy to use, but it assumes that one knows what one is looking for.  MarketPulse Pro can help the user find the right patterns based on historical statistics. All simulation tools here are for stock markets only.

 

Its interface includes three parts:  pattern setup, simulation output and file management.  A sample MarketPulse Pro interface is shown in Figure 5.1.  The usual procedures to use it are like this: set up a pattern that one wants to simulate, run the simulation back and forth to optimize it, then let MarketPulse monitor it and print out signals.

 

Figure 5.1 – The interface of MarketPulse Pro

 

Pattern Setup:

Pattern setup is a combination of General Filter and Digital Scan in MarketPulse.  One can use the Save button to save current patterns into a file.  Use Load and Delete buttons to review or delete previously saved patterns.  The Save button will overwrite old same-name files, if any, without warning.  The Load or Delete button will popup an interface to let the user choose which file to review or to delete.  The interface samples are shown in Figure 5.3.

 

The pattern setup text area is divided into two parts by [General Filter] and [MP Pattern], shown in Figure 5.2.  The user must use the same arrangement for every pattern setup file. They are the same as General Filter and Digital scan in MP, except that one can enter a specific stock symbol to only run using 'SList=IBM;' or 'slIST=ibm;', for example to enter IBM as run stock. The format is fixed but not capital sensitive, one can only replace string 'IBM' by any stock symbol. The other things one needs to note are that: 1,For the [MP Pattern] part, one must begin the pattern file in a fixed expression format:  nums=x; y=0; Where x is a positive integer the user can change, nums is a global variable to tell the program that a stock must have at least x bars to perform this pattern setup. The y is the signal, set to be 0 in the beginning (meaning no signal).  The whole setup program sets proper values for y under various conditions.  y=1 is for a long signal while y=-1 means a short sell signal. The variable for trigger-price is 'entryprice'. If it is not assigned a value in this pattern setup program, it will be automatically assumed as 'c(i)', the close price of the triggering bar. 2, For the [General Filter] part, we still use variables SInd and RmSI to choose sector index and KWIN and RmKW to filter conpany names. And they are also following the same rules as in MP. 3, Trading volume units for stocks in MP and PW is 100,000 shares for the daily data and 1000 shares for the intraday bars. 4, The Tick/Daily database button is only for variables in the [MP Pattern] part. They are all in lower cases. No related daily or intraday data in upper case are available for [MP Pattern] part.  For the [General Filter] part, upper case is for daily data variables and lower case is for intraday ones. One can use them at the same time to setup one's GF conditions.

 

The available variables are the same as in MarketPulse.  They are: o - open column; h - high column; l - low column; c - close column; v - volume column, i – the current bar index, ac3, c3m, ac6, c6m, ac12, c12m, ac25, c25m, ac75, c75m, ac250 c250m (close averages and their 1 bar momentums for 3, 6, 12, 25, 75 and 250 bars), av3, av6, av12, av25, av75, av250 (trading volume average for 3, 6, 12, 25, 75 and 250 bars), rsi7, rsi14, rsi30 (7, 14 and 30 bars' rsi) and Xi, Xi6, Xi12, Xi25, Xi75, Xi250 (S&P 500 index itself and averages for 6 12 25 75 and 250 bars) and Xi1m - mtm(Xi,1), Xi25m - (mtm(ma(SPXi25,2),1)), Xi75m - (mtm(ma(SPXi75,6),2)), Xi250m - (mtm(ma(SPXI250,15),5)).  Xr7, Xr14, Xr30 and Xr100 are 7, 14, 30 and 100 bars' rsi of S&P 500 index. To save space when to code ones pattern, we add i1 at the end of Xi to mean Xi(i-1), i.e. Xii1=Xi(i-1). The same for Xi6, Xi12, Xi25, Xi75, Xi250, Xi25m, Xi75m, Xi250m, Xr7, Xr14, Xr30, Xr100, rsi7, rsi14, rsi30, ac3, ac6, ac12, ac25, ac75, ac250, av3, av6, av12, av25, av75 and av250. For example, Xi250mi1=Xi250m(i-1). In MarketPulse, i is the index of the last bar, but here, it will be floated to point to the current bar during simulation loop. One can use any MATLAB and FPW functions.  There are couple pairs of  useful global variables for the user to control trading frequency and managing rules:  lastlow(2,3) and lastlop(2,3) - (second, third) last long signal’s date index and entry price, lastshw(2,3) and lastshp(2,3) - (second, third) last short signal’s date index and entry price;  StopfP - STOP value From Pattern and ObjefP - OBJEctive value From Pattern for the last signal, long or short. StopfP and ObjefP are always positive numbers, they mean the absolute stop or objective value from signal's entry price. Stopfp/Objefp can be used together with other simulation managing rules.  For example, 1) expression ‘i-lastlow>10’ means to take no more new long signals within 10 bars of the previous long signal bar; 2) Stopfp=abs(c(i)-c(i-1)) means the stop price is the net on signal day, i.e. trade will be stopped out whenever price is against the signal direction by the amount of signal day's absolute net.

Simulation Parameters and Managing Rules:

Before running a simulation, one needs to complete the parameter setup and managing rule sections, because the program will use these parameters to control the way to run and to output the results.  This part is located in the left lower corner, as shown in Figure 5.4.  The number in the Simulation Period indicates how many of the latest tick or daily bars to simulate against.  Holding length controls how many bars to hold a signal.  Because most simulation results are in data tables, we also call them simulation statistical results.  The number in the next three lines are for the V, P and VP buttons in the simulation output part to use when the program outputs V (volume in signal bar), P (entry price) and VP (volume in signal bar times entry price) statistical tables.  We will discuss them later. 

 

We introduce some simple optional managing rules for the forward simulation.  Compared to PatternWorkshop’s managing rules, here the rules are very simple.  They are the last two lines in the parameter setup part.  The left button on the second to last line controls whether to use money-managing rules as it shows.  The middle edit cell on the last line is the number for stop, and the right one is for objective.  They must be positive numbers.  The meaning of the numbers depends on what shows in the middle dropdown menu of the second to last line.  If it is "EntPrice", the numbers mean what percentage of the entry price; If it is "|Enp-c1|", the numbers mean what percentage of the absolute value of entry price minus one bar ago's price; if it is "Value", the numbers themselves are directly used as stop and objective in dollars, and if it is ‘Volatility’, then the stop and objective are what percentage of its x (x is the number showing in the dropdown menu next to it in the right.) bars volatility of one bar before the signal.

 

The right-side green RM button on the last line is used to simulate the latest simulation (Simu 1) with different Money Managing Rules currently showing in these two lines. It will force to set the whether to use money-managing rules button to 'Use'. It is tens times fast to run this RM operation than to Run a new simulation if one just needs to change ones simulated model with a different money managing rules. But if the trading signals are not independent each other in a model, one should use a thorough new Run instead of RM operation when to change the money managing rules, otherwise the output results might be wrong due to the characteristics of the simulated model. Finally, like Run and Load operations, RM operation results will be saved in Simu1 file to replace the original Simu1 file.

Pattern Simulation:

Upon finishing the pattern and parameter setup, one can start to simulate the pattern over all or one of AMX, NYSE or OTC stocks by clicking on the Run button in the pattern setup part.  The graphic output and all initial parameters will be showing in the upper output part, like the one shown in Figure 5.6. 

 

Figure 5.6 – Stock Simulation Output

 

From here one can do some optimizing work.  On the left side is the statistical data. On the right side is the cumulative net chart.  The two bottom lines on the right are filled with many interactive buttons used to find filters.  In the up line, one can enter range filters of volume, price and volume*price (the same meanings as in the parameter setup). The program will only choose trades in between the left and the right ones.  The first edit cell in the bottom line is the number for Simulation Period. One can now change it, but only to a smaller value each time. It means to output how many most recent bars' simulation results.  The DirectionButton on the left side rotates among all signals ‘+/-’, short only signals ‘- -’ and long only signals ‘+++’.  The two keys on DirectionButton's left are used to choose signal's begin-time, and the two on DirectionButton's right are used to decide signal's end-time. The keys with numbers on theirt dropdown menus mean how many bars to begin or to end a signal.  The ranges are from 0 (for the begin number) or 1 (for the end number) to the number entered into Holding length in the parameter setup part or the end number set during the last run, whichever smaller. During the first run of any model, the begin-pair keys are always assigned as 0/Close. And the other two end-pair are assigned as (Holding Length)/Close. The other two buttons, which are either ‘Close’ or ‘Open’, decide where exactly the signal to begin or to end.  The following V, P and VP push buttons are used to output V, P and VP statistical reports (shown in Tables 5.2 –5.4) based on all of the above conditions of volume, price, volume*price, historical data period, direction, statistical report length, the open or close status of its last line and any other conditions entered in the ANY cell (see next) to output the corresponding statistics.  The output tables are organized into several groups for V, P and V*P according to the two numbers entered in the parameter setup part.  The data area is the cumulative net. The first data column is for the trades whose V, P or V*P are less than the first number, and the rest of the columns are for the trades whose V, P or V*P are in a step of the second number, on the basis of their previous column. The small red M button is used to open a popup window to monitor the simulation when it is running.

 

The Any Cell

The biggest editable text box in the bottom line, called ANY cell, is used to enter any statement in the pattern setup text area.  The filter will keep only the signals with their ANY results being positive, i.e. >0.  The ANY statement can be divided into three categories:

  • 1. the original one, displaying ‘Enter any statement here’, will do nothing;
  • 2. beginning the statement with two capital letters ‘NO’ + some statements;
  • 3. Special cases, starting with 'DCR', 'qzi', 'TLR', 'NDN', 'NDU', 'VLI', 'VEM', 'spyCheck' and 'spychk'.
  • 4. any other case. 

In the second case, the program will not go to search the database to load data to calculate the ANY results.  It has already carried some data for each signal.  The available variables for one to use in this case are limited to these carried data, which are: enw – entry place index; enp- entry price, exp - exit price, endi- entry direction (long: +1, short: -1); sct - sector index, c0 – the signal bar’s close, i.e. the entry price; c1, c2 – the close of one or two bars ago; v0 – the signal bar’s trading volume; v1, v2 – the trading volume of one or two bars ago; r3, r6, r12 – last three, six and twelve bars’ average range, i.e. the volatility; v3, v6, v12, v25, v75, v250 – last three, six, twelve, twenty five, seventy five and 250 bars’ average trading volume, c3, c3m, c6, c6m, c12, c12m, c25, c25m, c75, c75m, c250, c250m, rsi7, rsi14, rsi30 and h0, l0, h3, l3, h6, l6, h12, l12, h25, l25, h75, l75, h250, l250, h500 and l500 – last three, six, twelve, twenty five, seventy five, 250 and 500 bars’ average closes and their 1 bar momentums, 7, 14 and 30 bars rsi, and current, 3, 6, 12, 25, 75 and 250 bars' high and low, and Xi, Xi6, Xi12, Xi25, Xi75, Xi250 (S&P 500 index itself and averages for 6 12 25 75 and 250 bars), Xr7, Xr14, Xr30, Xr100 (7, 14, 30, and 100 bars rsi of S&P 500 index) and , Xi25m - (mtm(ma(SPXi25,2),1)), Xi75m - (mtm(ma(SPXi75,6),2)), Xi250m - (mtm(ma(SPXI250,15),5)). In addition, several others about stock's basic information are also included: 'sct' for sector index, 'noe' for number of employee, 'ccode' and 'cap' (in $B, market capital at the triggering time). All averages are not including signal-bars.  If only ‘NO’ is entered, it will do nothing. Please note: Any more than one bar's data counts from one bar earlier, does not includes the current bar. lastbs - number of bars between buy-buy or short-short signals. Others (only available to selected users): mpind, stobi, hn, wr, tn, mynh, mydh, Xs, Xhh, Xh[l][0 3 6 12 25 75 250 500] and Xih[l][6 12 25 75 250].

 

For the third special cases, when

  • 1) starting with 'DCR' following with a positive number (<=0 or no number entered will re-assign as 1), the number is used as 'weigh' to Dcom1 when to do Dcom 2+1 operation, while Dcom2's weight is 1.
  • 2) starting with 'qzi' following a positive numbers separated by ';' (<=0 or no number entered will re-assign as 1), if only one number entered, the number will be used to replace any none-ones weights when to do reviewing operation Dcom1 or Dcom2. When paired numbers entered, will use the earlier one to replace the later one. When used together with others, always put 'qzi' in the first and separated by coma sign ','.
  • 3) starting with 'tlr' following one or two numbers, separated by coma or space. These two numbers will be used as the net range per share to show when and only when to click 'TradingLists' button. When only one number entered, it will be assigned as the low limit and the high limit assigned as positive 99999. When no number followed, range will be [-99999 99999].
  • 4) starting with 'spychk' or 'spycheck', spy price lines will be shown as background p in the output graph area. 'spyfilter' is still under testing.
  • 5) starting with 'NDN' following any or empty, it means no dollar neutral to output when and only click the 'Equity' button located at the lower right part.
  • 6) starting with 'NDU' following any or empty, it means the Dcom output results will not effect previous saved results inside DCOM1 and DCOM2, when and only click the 'Dcom' green button. It always starts first. Like 'NDUNO....'.
  • 7)  'VLI' following one or two number seperated by coma ',' sign is used to decide the position size when to run SIM1, SIM2, DCOM1, DECOM2 and DCOM2+1. The first number is the percentage of signaling day's trading volume. The second number is for the ceiling in 1000 shares, with default value of 1000 when omitted. For example, 'VLI5' means the trading size will be 5% of its signal day's volume with a ceiling of 1 million shares. 'VLI15,2000' means he trading size will be 15% of its signal day's volume with a ceiling of 2 million shares. It means to trade 1000 shares for each signal when no using VLI.
    When VLI's first number is entered a negative number, it means to simulate in dollar equal weighted, the number itself means how many thousand dollar to trade each signal. The meaning for the second VLI number is always the same as above: share limit.
    When used together with others, always put 'VLI' at the end.

 

For the 4th case, besides the variables for the second case, one can use any bar’s (better to keep to the most recent 100 bars.) open (o), high (h), low (l), close(c) and volume (v) by using the index i as the index of the signal bar.  Like what we've already used in many places, c(i-x) means the close of x bars ago relative to the signal bar.  It takes relatively longer time to use this format ANY statement because the program will go to search the database and evaluate the Any statement for each trades.  Note: The earlier a simulation was run, the slower this Any function will run. Because the program needs to load more old data, if the data is still kept. Remember that our databases are growing for each trading second. For intraday database simulation, one month after one run his/her simulation, it'd be better not to run Any operation for this case any more.

 

Clicking on the Run button here, the program outputs the new decomposed simulation output graphically. The Reset button is used to reset the window back to default numbers. Whenever the run button in the output part is used, a new decomposition simulation is saved.  And the next run will be based on the last decomposition simulation. The original simulation is used only in the beginning or after the reset button is used.  The program always keeps a copy of the last two original and the last two decomposition simulations.  Clicking on the graphic review and data transfer buttons in the right lower corner (shown in Figure 5.5), one can review, combine and transfer data between simulations.  In the same part, one can also get the trading list or daily equity report by clicking on TradingLists or Equity button. The Simulation Details button on the bottom line outputs the detailed forward statistics, a sample table is shown in Table 5.1.  The dollar value in the second title line represents the average price of all signals.  The second column in the data area is the cumulative net.  There are two data for each bar, one for the open time and one for the close time.  On the third row, UseRules-0 means no managing rules are used and UseRules-1 means some rules are used, and the rule details are listed in the rest of the same line.  Numbers after StopHit and ObjeHit are how many trades hit stops and objectives, if applied.
